Tag Description Example Code Notes
<header> Describes the content at the of the page <body>. Includes a logo, navigational links or a search bar. <!--Non Semantic HTML-->
<div id="footer">
<p>this is a footer</p>
</div>

<!--Semantic HTML-->
<footer>
<p>this is a footer</p>
</footer>		 Semantic HTML elements give developers an accurate view of what the element does or where it is placed based on the name of that element.
Semantic HTML introduces meaning to code. Before Semantic HTML the elements didn`t have any meaning as to what it does or what content goes in it. An element such as <div> was used as a general-purpose element to create things from headers to footers to articles. With Semantic HTML we were introduced to elements that tell developers and browsers exactly what it does and what content should go in it.
<nav> Encapsulates the page`s navigatioonal links. Often placed inside the <header> or <footer>.
<main> Encapsulates the main content of a page between the <header>, navigation and the <footer> areas.
<footer> Includes the page`s footer content at the botton of the <body>.
<section> Defines elements in a document, such as chapters, headings, or any other area of the document with the same theme. <section>
<!--defines theme-->
<h2>Top Sports league in America</h2>
<!--writes independent content relating to that theme-->
<article>
<p>One of the top sports league is the nba.</p>
</article>
</section>		 Generally <section> is used to define a theme for the webpage and <article> is used to write independent content for that theme.
But it does not mean, that <article> has to be used with <section>.
<article> Holds content that make sense on its own such as articles, blogs and comments.
<aside> Used to mark additional information that can enhance another element but isn`t required in order to understand the main content. <article>
<!--Main Content-->
</article>
<aside>
<!--Additional information-->
</aside>		 Usually, this information would be in a sidebar or a location where it doesn`t obstruct the main piece of content. For example an article, discusses how to take care of a dog and next to the article an ad would appear advertising a dog grooming product.
<video> Add videos to the website. <video> requires a closing tag. <video src="4kvideo.mp4">video not supported</video> Elemens all use the src attribute to link the source of the content.
<audio> Implement audio to the website. <audio> requires a closing tag. <audio src="koreanhiphop.mp3"></audio>
<embed> Used to implement any type of media. <embed> is a self-closing tag. <embed src="babyyoda.gif"/>
<figure> Encapsulate media such as inages, diagrams or code snippets. <figure>
<img src="qwerty.jpg">
<figcaption>The image shows the layout of a qwerty keyboard.</figcaption>
</figure>		 Normally used <figure> with its associated <figcaption> nested inside. This way, if position of media will change, the description will follow along with it.
<figcaption> Used to describe the media encapsulated within the <figure> element.
<adress> The <adress> element is like <time> in that it doesn`t deal with the overall structure of a document, but rather embellishes the parent <article> or <body> element with some metadata. It defines contact information for the author of the article or web page in question. <footer>
<p>This fake article was written by somebody at InternetingIsHard.com, which is a pretty decent place to learn how to become a web developer. This footer is only for the containing <article>element.</p>
<address>
Please contact <a href='mailto:xyz@example.com'>xyz</a> for questions about this article. </address>
</footer> 		<adress> should not be used for arbitrary physical addresses!
<time> The <time> element represents either a time of day or a calendar date. <header>
<h1>Semantic HTML</h1>
<p>By xyz. Published <time datetime='2017-1-3'>January 3rd</time></p>
</header>		 Providing a machine-readable date makes it possible for browsers to automatically link it to users` calendars and helps search engines clearly identify specific dates.
